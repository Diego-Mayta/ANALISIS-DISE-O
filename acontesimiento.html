Lista de Acontecimientos

Relación de todos los flujos de información entre el sistema SuperTicket.bo y las entidades externas identificadas en el Diagrama de Contexto.

N° Evento / Flujo Origen Destino Tipo Descripción
E01 Registro de nuevo evento Organizador Sistema Entrada El organizador ingresa datos del evento: nombre, fecha, lugar, precios, aforo, imagen, etc.
E02 Confirmación de evento creado Sistema Organizador Salida El sistema confirma que el evento ha sido registrado correctamente.
E03 Solicitud de compra de entrada Cliente Sistema Entrada El cliente selecciona evento, cantidad y tipo de entrada.
E04 Solicitud de pago Sistema Pasarela de Pago Salida Envío de datos de transacción a la pasarela (tarjeta, QR, etc.).
E05 Confirmación de pago Pasarela de Pago Sistema Entrada La pasarela notifica que el pago fue aprobado.
E06 Envío de entrada digital Sistema Servicio de Correo Salida Se genera PDF con código QR y se envía al cliente.
E07 Confirmación de entrega Servicio de Correo Sistema Entrada Notificación de que el correo fue enviado exitosamente.
E08 Consulta de reporte de ventas Organizador Sistema Entrada Solicitud de estadísticas y reportes del evento.
E09 Envío de reporte Sistema Organizador Salida El sistema entrega reporte en formato PDF o pantalla.
La Lista de Acontecimientos constituye la base para la descomposición funcional del sistema y el posterior desarrollo de los Diagramas de Flujo de Datos (DFD) de nivel 0 y 1.
